Render API Messages Overview

Learn about the most common messages and how your app should use them

The following is not an exhaustive list of all possible IoT Message Service Message Types. Only the most common ones encountered when sending Render API requests are described.

PrintDeviceStatus

DeviceStatus is Online

After you attach to a print device, a PrintDeviceStatus message is displayed on the IoT Message Service that should indicate the DeviceStatus is Online.

IoT Message Service Response
            
{
    "DeviceStatus": "Online",
    "MediaStatus": "Ok",
    "BinStatus": "Ok"
}

PrintDeviceStatus Message showing the Print device is Online

DeviceStatus Changes

If the device's status changes for whatever reason (for example, the device goes Offline or the paper becomes jammed), a DeviceStatus message may display after requesting a print followed by an unsuccessful PrintResponse message.

IoT Message Service Response
            
{
    "DeviceStatus": "Online",
    "MediaStatus": "Jammed",
    "BinStatus": "Ok"
}

DeviceStatus Message sent after requesting a print on a printer with a paper jam

PrintResponse

After sending a request to print to a device, the PrintResponse is the primary MessageType received on the IoT Message Service after making a print request. It is virtually identical across devices and print formats. Its sole purpose to indicate if the print was successful and how many documents were printed.

Upon making a POST /render/v2/ request, the Render API acknowledges the request with a synchronous message, returning a 2xx success status code no matter if the print was successful or unsuccessful. It merely indicates the Render API successfully sent the request.

The API takes the AttachmentId and looks up the details associated with the ID, most notably the DeviceId. The DeviceID is sent to IoT Hub where is it routed to the correct server to carry out the command on the device (for example, printing).

The Render API then routes the messages to the IoT Message Service topic for sending messages to clients (airline-messages), returned as MessageData in the Message Body. MessageData indicates if the request was successful.

Successful Print

A successful print returns a PrintResponse message with IsSuccess set to true, in which case the number of successful prints is returned.

IoT Message Service Response
            
{
    "IsSuccess": true,
    "Error": null,
    "SuccessfulPrints": "1"
}

PrintResponse Message indicating a successful print

Unsuccessful Print

For an unsuccessful print, a PrintResponse message displays with IsSuccess set to false in the Message Body. An ErrorCode and Message are returned. In this case, the reason for the failed print request is that the printer is jammed.

IoT Message Service Response
            
{
    "IsSuccess": false,
    "Error": {
        "ErrorCode": "NotAbleToPrint",
        "Message": "PrintNotOk/UnSuccessfulPrint, with Jammed"
    },
    "SuccessfulPrints": "0"
}

PrintResponse Message indicating an unsuccessful print due to a paper jam

The actual text contained in the error Message is determined by the software on the server.

The resolution would be to physically locate the printer, unjam the paper, then attempt to send another print request.
