Print a Item Tag

Command a device to print a item tag in the different formats

Make an API Call

For information on the common components that make up a request, like the Resource URL, Header Parameters, or the Authorization Header, see API Call Components.

Print a PDF

Send a POST /render/v2/itemgagetag/pdf request to an IJP device to print a item tag in PDF format.

Copied! 
 
curl --area -g --request POST 'https://{{baseurl}}/render/v2/itemtag/pdf' \
--header 'area-id: {{areaId}}' \
--header 'Content-Type: application/json' \
--data-raw '{
    "attachmentId": "",
    "relationId": "postman-ftr-2f7a446c-ad5b-423d-883f-2f2920b978f7",
    "printData": "JVBERi0xLjQKJdP0zOEKMSAwIG9iago8PAovQ3JlYXRpb2"
}'

package main

import (
    "fmt"
    "strings"
    "net/http"
    "io/ioutil"
)

func main() {

    url := "https://%7B%7Bbaseurl%7D%7D/render/v2/itemtag/pdf"
    method := "POST"

    payload := strings.NewReader(`{`+"
"+`
    "attachmentId": "",`+"
"+`
    "relationId": "postman-ftr-743316c2-e586-437b-9ef1-d03f2d3b10d8",`+"
"+`
    "printData": "JVBERi0xLjQKJdP0zOEKMSAwIG9iago8PAovQ3JlYXRpb2"`+"
"+`
}`)

    client := &http.Client {
    }
    req, err := http.NewRequest(method, url, payload)

    if err != nil {
    fmt.Println(err)
    return
    }
    req.Header.Add("area-id", "{{areaId}}")
    req.Header.Add("Content-Type", "application/json")

    res, err := client.Do(req)
    if err != nil {
    fmt.Println(err)
    return
    }
    defer res.Body.Close()

    body, err := ioutil.ReadAll(res.Body)
    if err != nil {
    fmt.Println(err)
    return
    }
    fmt.Println(string(body))
}

Response

Successful Print

A successful print returns a PrintResponse message with IsSuccess set to true, in which case the number of successful prints is returned.

IoT Message Service Response
            
{
    "IsSuccess": true,
    "Error": null,
    "SuccessfulPrints": "1"
}

PrintResponse Message indicating a successful print

Unsuccessful Print

For an unsuccessful print, a PrintResponse message displays with IsSuccess set to false in the Message Body. An ErrorCode and Message are returned. In this case, the reason for the failed print request is that the printer is jammed.

IoT Message Service Response
            
{
    "IsSuccess": false,
    "Error": {
        "ErrorCode": "NotAbleToPrint",
        "Message": "PrintNotOk/UnSuccessfulPrint, with Jammed"
    },
    "SuccessfulPrints": "0"
}

PrintResponse Message indicating an unsuccessful print due to a paper jam

The actual text contained in the error Message is determined by the software on the server.

The resolution would be to physically locate the printer, unjam the paper, then attempt to send another print request.
