GateAccess API

Learn how to interact with a gate device to perform boarding processes

The GateAccess API endpoints allow your airline backend to interact with Magstripe Readers (BGR/BCR) and Self-Boarding Gates (SBG).

Endpoints
/gateaccess/v2/opengate
gateaccess/v2/acceptconsumer
/gateaccess/v2/rejectpassenger
/gateaccess/v2/closegate

The supported devices at present are the following:

Acronym Definition Description
BGR Magstripe Reader Considered part of an agent-driven boarding process not involving a gate whereby the agent scans a passenger's boarding pass. If accepted, the passenger may move past the agent and board the flight.
BCR Boarding Card Reader Functions the same as a BGR. For this documentation, the use of the acronym BCR is synonymous with BGR.
SBG Self-Boarding Gate Considered part of a passenger-driven boarding process involving a gate whereby the passenger scans his or her boarding pass at a gate. If accepted, the gate opens, and the passenger can pass through the gate.
