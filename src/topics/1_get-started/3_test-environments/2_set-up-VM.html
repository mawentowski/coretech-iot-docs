Citrix Virtualized CoreTech IoT Environment

Set up a virtual machine for application development

In addition to using the devices included in the CoreTech IoT bundle, you may also interact with CoreTech APIs using a Device simulator in a virtualized environment configured with the CoreTech platform. The Device Simulator hosts "fake" devices that behave like the real ones included in the bundle.

It is recommended that you download the Citrix Workspace application to access CoreTech's Citrix NetScaler Gateway.

Install Citrix Workspace

VM
  1. Download the latest version of the Citrix Workspace app.
  2. Download and open the installer.
  3. Select all default settings.
    • The installation completes, with the option to Add Account.
  4. Click Add Account.
    • The Citrix Workspace displays.

Log in to Citrix Workspace

VM
  1. Enter and submit the Store URL provided to you by CoreTech.
  2. Enter your Citrix Workspace Username and Password provided by CoreTech.
  3. Click Sign In.

Optional: View Primary Account

VM

After signing in for the first-time, Store Service has now been added as your primary account.

To view accounts:

  1. Click the drop-down arrow to the left of the search bar in the application header and select Accounts.
    • The (Primary) account Store Service is visible.
    • You may also Add or Remove accounts from this window.

  2. Click Ok to close the window.

Add VM to Workspace

VM

If the VM you are assigned is not visible from the home page, you may add it.

To add a VM to your workspace:

  1. Click on the + icon on the left side of the screen.
  2. Click the VM you have been assigned.
    • A VM icon transitions onto your workspace that allows quick access to that VM.
    • If you click + again, you will notice a green check mark displays next to the VM to indicate it has been added.

Log into the VM

VM

Login to the virtual machine using the credentials provided to you as part of registration.

Copying and pasting information into the VM is not allowed for security reasons. You must hand type all information. If you need assistance making something available on the desktop, contact techsupport@techwritex.com.

To log in to the VM:

  1. Click the quick access link to the VM you have been assigned from your workspace.
    • If you see an error stating "The resource is unavailable currently", refer to the Troubleshooting section. troubleshooting an unavailable VM.
  2. Enter your VNC Password provided by CoreTech.
    • If successful, the Windows lock screen displays.
  3. Right-click the blue header of the VNC Window and select Send Ctrl+Alt+Del.
    • A VM Windows Login screen displays.
  4. Enter and submit the Windows Credentials that you have been provided by CoreTech.
    • The Device View opens and displays the status of the devices.

Add a Device

VM

Devices must be added using the Device Simulator before they can interact with CoreTech APIs.

To add a device:

  1. Double-click the Device Simulator icon screen on Windows desktop.
    • The Device Simulator displays.

  2. Click Add Device.

    3. For demonstrative purposes, you will be adding an MFPStandard device. However, note that there are many other devices available.

  3. Click the Device Type drop-down and select MFP Standard.

  4. Leave all default values the same for the selected MFP Standard device.

  5. Click Add Device.
    • The device now displays in Device Simulator and is available to Run.

  6. On the device, click Options, then Settings.

    • An MFP Settings window displays.
    • You may modify several fields that enable changes in the behavior of the device. However, the default values should be enough for your initial development.
  7. Click Save and Close in the MFP Settings window.

Run the Device

VM
  1. On the device, click Run.
    • The MFP Simulator page displays, and the Session Log updates as the device is activated.

  2. From Windows desktop, return to the Device View and observe that the MFP device status is now showing in green as Online.

    3. After having successfully added and run a device in the Device Simulator, you are now ready to interact with the device using the CoreTech APIs.

In Conclusion

Repeat the steps above for any other device type. Many devices can run at the same time using the Device Simulator.

All the default values should be used when adding and running the device, unless otherwise stated.

Continue to Get Started Using CoreTech APIs to interact with the device you just added.
