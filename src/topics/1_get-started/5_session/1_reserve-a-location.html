Reserve a Location

Learn how to reserve a location to interact with its devices

The POST /site/v2/reservations request limits use of a location to the application backend with a specific customer code that made the request. The application backend must reserve the location to make use of its devices.

Flex knows the customer code of the requester because it is encoded in the access token.

Make an API Call

To successfully make the request, you must pass the LocationId in the Request Body of the location you wish to reserve.

Copied! 
 
curl --location -g --request POST 'https://{{baseurl}}/site/v2/reservations' \
--header 'location-id: {{locationId}}' \
--header 'Content-Type: application/json' \
--data-raw '{
    "LocationId": "{{locationId}}"
}'

)

func main() {

    url := "https://%7B%7Bbaseurl%7D%7D/site/v2/reservations"
    method := "POST"

    payload := strings.NewReader(`{`+"
"+`
    "LocationId": "{{locationId}}"`+"
"+`
}`)

    client := &http.Client {
    }
    req, err := http.NewRequest(method, url, payload)

    if err != nil {
    fmt.Println(err)
    return
    }
    req.Header.Add("location-id", "{{locationId}}")
    req.Header.Add("Content-Type", "application/json")

    res, err := client.Do(req)
    if err != nil {
    fmt.Println(err)
    return
    }
    defer res.Body.Close()

    body, err := ioutil.ReadAll(res.Body)
    if err != nil {
    fmt.Println(err)
    return
    }
    fmt.Println(string(body))
}

Sample POST /site/v2/reservation request containing the LocationId passed from the client app

Response

A successful request returns a 2xx success status code with the JSON Response Body containing the reservationId. You must include this ID later in connection requests to devices at the reserved location.

Response
        
[
    {
        "reservationId": "868e5715-2c82-426e-b7da-10e208dd865d",
        "customerCode": "ZZ",
        "locationId": "A-ZZU-MZUZUAIRPORT-ZZU1CKB004",
        "expiryTime": "2021-11-24T19:26:30.846778+00:00"
    }
]

Sample Response Body for POST /site/v2/reservation request containing the reservationId

The Response Body contains informational properties such as the customerCode encoded in the access token and locationID passed in the Request Body. The property expiryTime indicates when the reservation is due to expire.

It is important that application backends keep track of when a location reservation is about to expire so it can request a new reservation if continued access is desired.

Reservation Error

Now that the location is reserved by an customer code, any request made by another airline will be rejected. This will occur until the reservation's expiryTime has been reached or the airline who made the reservation has sent a DEL /site/v2/reservation to delete the reservation.

The following error displays when attempting to reserve a location that is already reserved:

Error
            
{
    "type": "https://tools.ietf.org/html/rfc7231#section-6.5.1",
    "title": "Bad Request",
    "status": 400,
    "detail": "Location reserved by other clients.",
    "traceId": "00-25dd79293ba71f479ce30da826eecd09-3aa97e2de755f540-00"
}

Sample 400 Bad Request Error Message when a location is reserved by another airline

Sending a DEL /site/v2/reservations/{reservation_id} request returns a 2xx success status code and deletes a reservation if one exists.

Deleting Reservations

It is important to delete the reservation once done with a location because an active reservation prevents other application backends from using the location. This is achieved by sending a DEL /site/v2/reservations/{reservation_id} request, passing in the reservation_id of the reservation to be deleted as a path parameter.
