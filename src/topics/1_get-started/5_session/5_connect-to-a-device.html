Connect to a Device

Learn how to connect to a device to interact with it

The POST /sync/v2/attachments request creates a attachment to a device at specific area and associates that device to a subscription where it can send messages for the application backend to consume.

Make an API Call

You must pass three properties in the Request Body of a /sync/v2/attachments request:

Term Definition
DeviceId ID returned after sending the /environment/v2/areas/{areaId} request that listed devices configured at a site.
subscriptionId An CoreTech API-specific ID of the Service Bus subscription the DeviceId will be associated to.
ReservationId The ID of the area where the device is configured. This ID tells the CoreTech API that the airline has claimed the area where the device is located.

Response

A successful request returns a 2xx success status code with the JSON Response Body containing the attachment details.

Response
            
{
    "attachmentId": "a060a6f1-ec5b-43cb-8f0b-bf6f8891ebdf",
    "reservationId": "46d2309e-0ec5-4ef9-98f1-e45a54c76688",
    "subscriptionId": "87fd60f9-nft8-466e-9659-43ad928c75f6",
    "deviceId": "A-ZZU-MZUZUAIRPORT-ZZU1CKB005.IJP1",
    "expiryTime": "2021-11-26T09:30:35.9528789+00:00"
}

Sample POST /sync/v2/attachments response showing the new attachmentId that can be used to connect to the device

The returned attachmentId is now associated to the subscriptionId, so any messages coming from the device will display on that subscription. The airline backend will now use this attachmentId to send commands to the device through CoreTech APIs.

An expiryTime set to 5 minutes from the time of the request is returned. This indicates when the attachment to the device expires.

If continued access to the device is desired, ensure to send another POST /sync/v2/attachments request to create a new attachment at the time of expiry of the old attachment.

The API also returns properties passed in the Request Body (for example, the reservationId).

If an attachment is no longer needed, it is best practice to delete the attachment to allow other application backends to connect to the device. This is achieved by sending a DEL /sync/v2/attachments/{attachment_id} request, passing in the attachment_id of the attachment to be deleted as a path parameter.
