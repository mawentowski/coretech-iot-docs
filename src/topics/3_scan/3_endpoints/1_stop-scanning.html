Stop Scanning

Stop the scanning process on an active scan device
Send a POST /scan/v2/stopscanning request to release a device from exclusive access so other airline backends can claim the device and use it to read travel documents.

import (
  "fmt"
  "strings"
  "net/http"
  "io/ioutil"
)

func main() {

  url := "https://%7B%7Bbaseurl%7D%7D/scan/v2/stopscanning"
  method := "POST"

  payload := strings.NewReader(`{`+"
"+`
    "attachmentId": ""`+"
"+`
}`)

  client := &http.Client {
  }
  req, err := http.NewRequest(method, url, payload)

  if err != nil {
    fmt.Println(err)
    return
  }
  req.Header.Add("area-id", "{{areaId}}")
  req.Header.Add("Content-Type", "application/json")

  res, err := client.Do(req)
  if err != nil {
    fmt.Println(err)
    return
  }
  defer res.Body.Close()

  body, err := ioutil.ReadAll(res.Body)
  if err != nil {
    fmt.Println(err)
    return
  }
  fmt.Println(string(body))
}

Sample POST /scan/v2/stopscanning request to releaese an HHS device from exclusive access and stop the scanning process

Request Body

To successfully make the request, you must include the attachmentId of the device you wish to release.

Request Body
            
{
    "attachmentId": "{{hhs_attachment_id}}"
}

Sample Request Body for POST /scan/v2/stopscanning request that provides a attachmentId of the device to release

Response

The Scan API acknowledges the request with a synchronous message, returning a 2xx success status code (as seen in a tool like Postman).

An asynchronous ScanDeviceStatus message is sent to all subscribers with an active attachment to the scan device to indicate the device is no longer exclusively being used ("DeviceInUse": "false") and is available to successfully receive a /scan/v2/startscanning request.

Service Bus Message
            
{
    "DeviceStatus": "Online",
    "DeviceInUse": "false"
}

DeviceStatus Service Bus Message Body indicating the device is no longer 'in use' and is available to be claimed
