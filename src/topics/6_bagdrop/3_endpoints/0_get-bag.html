Initiate a dispatch process

Send a POST /dispatchitem/v2/getitem request to initiate a dispatch process and open the door (if there is a door) of the GRC so the consumer can place their item on the belt.

Sample POST /dispatchitem/v2/getitem request to initiate a dispatch process.

Request Body

To successfully make the request, you must include the attachmentId of the dispatchitem device.

Request Body
            
{
    "attachmentId": "{{grc_attachment_id}}",
    "relationId": "postman-ftr-{{$guid}}"
}

Sample Request Body for a POST /dispatchitem/v2/getitem request that provides a attachmentId of the dispatchitem device

Response

Upon making a POST /dispatchitem/v2/getitem request, the Dispatch API acknowledges the request with a synchronous message returns a 2xx success status code (as seen in a tool like Postman).

The dispatch process is initiated, and the door to the GRC opens (if there is a door) so the consumer can place the item on the belt.

A GetItemResponse message displays on the Service Bus, acknowledging that the request to initiate a dispatch process was received

Baggage information may be collected by the GRC and displayed in a BagPlaced message on the Service Bus.
