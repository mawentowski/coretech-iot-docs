Process Bag

Process a bag so it continues the bagdrop process

After the passenger places a bag on the belt, the bag's measurements are taken, and a settled weight is calculated. Send a POST /bagdrop/v2/processbag request to close the door and move the bag from the Entry position to the Process position.

Make an API Call

For information on the common components that make up a request, like the Resource URL, Header Parameters, or the Authorization Header, see API Call Components.

/bagdrop/v2/processbag request to close the door and move the bag along the belt.

Request Body

To successfully make the request, you must include the connectionId of the bagdrop device.

Request Body
            
{
    "connectionId": "{{sbd_connection_id}}",
    "correlationId": "postman-ftr-{{$guid}}"
}

Sample Request Body for a POST /bagdrop/v2/processbag request that provides a connectionId of the bagdrop device

Response

Upon making a POST /bagdrop/v2/process request, the Bagdrop API acknowledges the request with a synchronous message returns a 2xx success status code (as seen in a tool like Postman).

A ProcessBagResponse message displays on the Service Bus, acknowledging that the request to process the bag was received.

A BagChanged message displays on the Service Bus and is identical to the BagPlaced message and displays baggage information collected through a measurement. The property Position changes from Entry to Process.
