Cancel Item

Cancel a dispatch process so an attendant can attempt to resolve an issue with the item

If an issue arises when the item is in the Process position, and the customer cannot to resolve the issue, send a POST /dispatch/v2/cancelitem request to open the door and return the item to the Entry position where the attendant can resolve the issue.

Make an API Call

For information on the common components that make up a request, like the Resource URL, Header Parameters, or the Authorization Header, see API Call Components.

Sample POST /dispatch/v2/cancelitem request to return the item to where the attendant can resolve the issue.

Request Body

To successfully make the request, you must include the attachmentId of the dispatch device.

Request Body
            
{
    "attachmentId": "{{grc_attachment_id}}",
    "relationId": "postman-ftr-{{$guid}}"
}

Sample Request Body for a POST /dispatch/v2/cancelitem request that provides a attachmentId of the dispatch device

Response

Upon making a POST /dispatch/v2/cancelitem request, the Dispatch API acknowledges the request with a synchronous message returns a 2xx success status code (as seen in a tool like Postman).

A AbortItemResponse message displays on the Service Bus, acknowledging that the request to return the item was received.

A BagChanged message displays on the Service Bus and is identical to the BagPlaced message and displays item information collected through a measurement. The property Position changes from Process to Entry.
