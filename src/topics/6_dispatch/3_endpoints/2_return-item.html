Return Item

Return the item to the customer to resolve the issue

If an issue arises when the item is in the Process position, and the customer can resolve the issue, send a POST /dispatch/v2/returnitem request to open the door and return the item to the Entry position where the customer to resolve the issue.

Make an API Call

For information on the common components that make up a request, like the Resource URL, Header Parameters, or the Authorization Header, see API Call Components.

Sample POST /dispatch/v2/returnitem request to return the item to the Entry position.

Request Body

To successfully make the request, you must include the attachmentId of the dispatch device.

Request Body
            
{
    "attachmentId": "{{grc_attachment_id}}",
    "relationId": "postman-ftr-{{$guid}}"
}

Sample Request Body for a POST /dispatch/v2/processitem request that provides a attachmentId of the dispatch device

Response

Upon making a POST /dispatch/v2/returnitem request, the Dispatch API acknowledges the request with a synchronous message returns a 2xx success status code (as seen in a tool like Postman).

A AdjustItemResponse message displays on the IoT Message Service, acknowledging that the request to return the item was received.

A ItemChanged message displays on the IoT Message Service and is identical to the ItemPlaced message and displays item information collected through a measurement. The property Position changes from Process to Entry.
